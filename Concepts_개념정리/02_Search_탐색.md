# 탐색 (Search Algorithm)

## 1. 개요
데이터 집합에서 특정 값을 가진 원소를 찾아내는 기법입니다. 데이터의 정렬 여부에 따라 효율성이 크게 달라집니다.

## 2. 주요 알고리즘 분석

### 2.1. 이진 탐색 (Binary Search)
- **전제 조건**: 데이터가 반드시 **정렬되어 있어야 함**.
- **방식**: 탐색 범위를 절반씩 좁혀가며 데이터를 찾는 분할 정복 기반 알고리즘.
- **시간 복잡도**: $O(\log n)$

### 2.2. 순차 탐색 (Sequential Search)
- **전제 조건**: 없음 (정렬되지 않은 데이터에서도 가능).
- **방식**: 처음부터 끝까지 하나씩 비교.
- **시간 복잡도**: $O(n)$

---

## 3. 시험 포인트 및 팁
- 이진 탐색의 **반복문 vs 재귀** 구현 방식의 차이점 이해.
- 탐색 범위가 매우 클 때(예: 억 단위)는 반드시 $O(\log n)$ 알고리즘을 고려해야 함.
- 삽입/삭제가 빈번한 데이터셋에서는 정렬 상태를 유지하는 비용과 탐색 비용의 트레이드오프를 고려함.
