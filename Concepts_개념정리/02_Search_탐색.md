# 02. 탐색 (Search Algorithm) - 중간고사 필승 요약

## 💡 Why? (왜 배우는가?)
- **데이터 활용**: 정렬된 데이터를 단순히 쌓아두는 것이 아니라, 원하는 정보를 빛의 속도로 찾아내기 위함입니다.
- **결정 문제 해결**: 최적화 문제를 "이 조건을 만족하는 값이 있는가?"라는 결정 문제로 바꿔 이진 탐색으로 해결(Parametric Search)할 수 있습니다.

---

## 🚀 핵심 알고리즘 1줄 암기 포인트

1. **순차 탐색 (Sequential Search)**: "그냥 처음부터 끝까지 다 보기! 정렬 안 돼 있어도 되지만 $O(n)$이라 느림."
2. **이진 탐색 (Binary Search)**: "**정렬 필수!** 업다운 게임처럼 범위를 절반씩 뚝딱! $O(\log n)$의 기적."

---

## 📊 시험 대비 비교표 (무조건 암기)

| 알고리즘 | 전제 조건 | 시간 복잡도 | 장점 | 단점 |
| :--- | :---: | :---: | :--- | :--- |
| **Sequential** | 없음 | $O(n)$ | 구현이 매우 쉬움 | 데이터가 많으면 매우 느림 |
| **Binary** | **정렬 상태** | $O(\log n)$ | 매우 빠름 (억 단위도 가능) | 정렬 비용이 발생함 |

---

## ❓ 시험 빈출 Q&A

**Q1. 이진 탐색을 적용하기 위해 반드시 선행되어야 하는 조건은?**
- **A**: 데이터가 특정 기준(오름차순/내림차순)으로 **정렬(Sorted)**되어 있어야 합니다.

**Q2. 이진 탐색의 탐색 범위가 $N$일 때, 최대 비교 횟수는?**
- **A**: $\log_2 N + 1$회 (정확히는 $\lceil \log_2 N \rceil$).

**Q3. 이진 탐색 구현 시 `mid` 계산에서 발생할 수 있는 잠재적 오류는?**
- **A**: `(low + high) // 2` 계산 시 `low + high` 값이 프로그래밍 언어의 정수 범위를 초과(Overflow)할 수 있음. 안전하게 `low + (high - low) // 2`로 쓰기도 함.
