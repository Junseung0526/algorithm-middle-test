# 정렬 (Sorting Algorithm)

## 1. 개요
데이터를 특정한 기준(오름차순, 내림차순 등)에 따라 나열하는 기법으로, 효율적인 탐색과 데이터 처리를 위한 필수 과정입니다.

## 2. 주요 알고리즘 분석

### 2.1. 퀵 정렬 (Quick Sort)
- **방식**: 피벗(Pivot)을 기준으로 데이터를 두 그룹으로 분할하여 재귀적으로 정렬하는 분할 정복 알고리즘.
- **시간 복잡도**: 
    - 평균: $O(n \log n)$
    - 최악: $O(n^2)$ (피벗이 항상 최소/최대값으로 선택될 경우)
- **시험 포인트**: 
    - 피벗 선정 전략: 중앙값 선택(Median-of-Three) 등으로 최악의 경우 방지.
    - 불안정 정렬(Unstable Sort): 동일한 값의 순서가 바뀔 수 있음.

### 2.2. 합병 정렬 (Merge Sort)
- **방식**: 리스트를 최소 단위까지 분할한 후, 정렬하며 다시 합치는 분할 정복 알고리즘.
- **시간 복잡도**: $O(n \log n)$ (항상 보장)
- **시험 포인트**: 
    - 공간 복잡도: 정렬 과정에서 추가적인 메모리($O(n)$)가 필요함.
    - 안정 정렬(Stable Sort): 동일한 값의 기존 순서가 유지됨.

---

## 3. 정렬 알고리즘 비교 (요약)

| 알고리즘 | 평균 시간 복잡도 | 최악 시간 복잡도 | 공간 복잡도 | 안정성 |
| :--- | :--- | :--- | :--- | :--- |
| 퀵 정렬 | $O(n \log n)$ | $O(n^2)$ | $O(\log n)$ | Unstable |
| 합병 정렬 | $O(n \log n)$ | $O(n \log n)$ | $O(n)$ | Stable |
| 선택 정렬 | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Unstable |
| 삽입 정렬 | $O(n^2)$ | $O(n^2)$ | $O(1)$ | Stable |
