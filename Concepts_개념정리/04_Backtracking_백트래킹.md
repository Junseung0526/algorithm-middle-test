# 백트래킹 (Backtracking)

## 1. 개요
해를 찾는 과정에서 막히면 뒤로 돌아가서 다시 해를 찾는 기법입니다. 주로 모든 경우의 수를 탐색해야 할 때 사용하지만, 효율성을 위해 불필요한 경로는 사전에 차단합니다.

## 2. 핵심 개념
- **상태 공간 트리 (State Space Tree)**: 문제의 해를 찾는 과정을 트리 구조로 시각화한 것.
- **유망성 (Promising)**: 특정 노드가 해를 포함할 가능성이 있는지 판단하는 기준.
- **가지치기 (Pruning)**: 유망하지 않은 노드는 탐색을 중단하고 부모 노드로 돌아가는 행위.

## 3. 대표적인 문제
- **N-Queens**: N x N 체스판에 퀸 N개를 서로 공격할 수 없게 배치하는 문제.
- **암호 만들기**: 조건을 만족하는 순열/조합 생성 문제.
- **미로 찾기**: 경로가 막히면 이전 갈림길로 돌아가 다른 경로를 찾는 방식.

---

## 4. 시험 포인트 및 팁
- **DFS(깊이 우선 탐색)**와의 차이점: DFS는 모든 노드를 방문하는 것이 목적이지만, 백트래킹은 가지치기를 통해 방문 횟수를 최소화함.
- 시간 복잡도는 일반적으로 지수 시간(Exponential)을 가지므로 데이터 규모가 작을 때 주로 사용됨.
